Časté dotazy

  • What technologies are used in the book?
    All the sample pages are Java servlets of a Maven project that runs as a web application on a Jetty container, but the WAR web archive can also run on Tomcat and other containers. The Element class from the free library is used to eliminate text strings with HTML code. The source code for the examples can be viewed here. Examples require Java version 17 LTS to run, which is the latest stable version of Java with long-term support (at the time the book was published).
  • Isn't using the object model of a web page complicated?
    I believe that anyone who wants to learn an object language should also experience working with an object model, and building an HTML model seems like a good opportunity to do so.
  • Isn't using the Element class complicated?
    Once you get the root element, you only need to know (in principle) three methods from that class to build any web page:
    1) setAttribute() - this method assigns an attribute with a value to the current element,
    2) addElement() - the method adds a new element to the object with the name of the first argument and returns its instance,
    3) addText() - the method adds text to the element body and handles the writing of special characters; there is an alternative method addRawText().
  • Isn't the web page data model memory intensive?
    The Element class is really just an envelope over an object of type PrintWriter, so any text written to the element is sent directly to the writer. With this approach, even pages of significant size can be built efficiently. However, the full memory model can be activated in the configuration if needed.
  • Why aren't the examples built on JSP or JSF?
    The goal of the book is not to teach the templating system, but to work with objects. In addition, opinions on the merits of JSP technology and its successor JSF tend to be highly controversial.
  • Could the book's concept lead to cargoclutic programming (coding without understanding)?
    The aim of the book is to demonstrate the meaning, principles and context of the programming constructs used. However, I believe that a beginner does not need to know all the details right away. Extreme attention to detail will unnecessarily discourage some beginners, and this is not just true of programming. Selected topics include links to Internet resources for further study.
  • Isn't mastering all of the technologies used beyond the capabilities of a beginner?
    It's true that web pages are described in HTML with CSS, and you need to use Maven to run the application (in a web container). However, the tools of the Maven project are primarily used as a resource here, and the same is true of the web container. I have tried to keep the explanation of HTML with CSS (including the techniques used) to the minimum necessary. The downloaded project contains a script to run the entire project and includes Maven itself. Apart from the Java environment, nothing else needs to be installed.
