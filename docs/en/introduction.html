Introduction

I would like to present the original Java textbook. I would like to thank the Department of Theoretical Computer Science at the FIT CTU for their help with proofreading. The book in PDF format has 154 A5 pages and contains over 110 colour code samples, illustrations, graphs and screenshots. The interpretation is primarily intended for high school students, but many more experienced readers will undoubtedly find inspiration there as well.
What makes this textbook different from others? The short answer is that the book is exceptional in its examples. In fact, the tutorial code is built on a Java servlet project that can be easily run in a Jetty container with graphical output to a regular web browser. However, because the goal of the tutorial is not to educate you about a templating system, the structure of the web pages is also built using an object model that is responsible for the correct formatting of the web page itself. I have no doubt that it is useful to learn how to work with an object model when teaching an object-oriented language, and building an HTML model seems like a good opportunity to do so. Even the AJAX proof-of-concept demonstrations are built on Java code so that the reader is not distracted unnecessarily by learning a scripting language.
The first part of the book covers more of the basics of computer science, data types, object-oriented programming concepts, web technologies, and even some object classes from the standard Java library. The second part guides the reader through an explanation of the programming language against the background of the attached project. The explanation of new topics always comes with new code for additional servlets whose programming resources are gradually expanding. This is perhaps all that is essential to the book. So if someone is looking for a quick beginner's guide to Java and related web technologies, this book is probably unrivalled (on the Czech market). However, if someone expects a detailed description of the Java language with examples dependent on the Swing graphical library, I would recommend looking for a different textbook.
